## <a name="summary"></a><span data-ttu-id="9a813-282">Souhrn</span><span class="sxs-lookup"><span data-stu-id="9a813-282">Summary</span></span>

<span data-ttu-id="9a813-283">Při definování schématu pro tabulku, Microsoft SQL Server umožňuje zahrnutí počítané sloupce.</span><span class="sxs-lookup"><span data-stu-id="9a813-283">When defining the schema for a table, Microsoft SQL Server allows the inclusion of computed columns.</span></span> <span data-ttu-id="9a813-284">Jedná se o sloupce, jejichž hodnoty se vypočítají z výraz, který obvykle odkazuje na hodnoty z ostatních sloupců ve stejném záznamu.</span><span class="sxs-lookup"><span data-stu-id="9a813-284">These are columns whose values are calculated from an expression that usually references the values from other columns in the same record.</span></span> <span data-ttu-id="9a813-285">Od hodnoty pro počítané sloupce jsou založené na výrazu, jsou jen pro čtení a nelze jí přiřadit hodnotu v `INSERT` nebo `UPDATE` příkaz.</span><span class="sxs-lookup"><span data-stu-id="9a813-285">Since the values for computed columns are based on an expression, they are read-only and cannot be assigned a value in an `INSERT` or `UPDATE` statement.</span></span> <span data-ttu-id="9a813-286">Vzniká problémy při použití počítaný sloupec v hlavním dotazu TableAdapter, která se pokusí automaticky generovat odpovídající `INSERT`, `UPDATE`, a `DELETE` příkazy.</span><span class="sxs-lookup"><span data-stu-id="9a813-286">This introduces challenges when using a computed column in the main query of a TableAdapter that tries to automatically generate corresponding `INSERT`, `UPDATE`, and `DELETE` statements.</span></span>
