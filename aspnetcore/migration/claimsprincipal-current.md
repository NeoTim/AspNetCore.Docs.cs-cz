<span data-ttu-id="c120b-112">Pochopit druhy problémů vlákno může způsobit statické členy v asynchronní scénáře, zvažte následující fragment kódu:</span><span class="sxs-lookup"><span data-stu-id="c120b-112">To understand the sorts of problems thread static members can lead to in asynchronous scenarios, consider the following code snippet:</span></span>

<span data-ttu-id="c120b-113">Předchozí sady ukázka kódu `Thread.CurrentPrincipal` a ověří jeho hodnotu před a po čeká na asynchronní volání.</span><span class="sxs-lookup"><span data-stu-id="c120b-113">The preceding sample code sets `Thread.CurrentPrincipal` and checks its value before and after awaiting an asynchronous call.</span></span> <span data-ttu-id="c120b-114">`Thread.CurrentPrincipal` je specifická pro *vlákno* na kterém je nastavená a metoda je pravděpodobně pokračovat v provádění v jiném podprocesu po await.</span><span class="sxs-lookup"><span data-stu-id="c120b-114">`Thread.CurrentPrincipal` is specific to the *thread* on which it's set, and the method is likely to resume execution on a different thread after the await.</span></span> <span data-ttu-id="c120b-115">V důsledku toho `Thread.CurrentPrincipal` je přítomen, když je nejdřív zaškrtnuta možnost, ale má hodnotu null po volání `await Task.Yield()`.</span><span class="sxs-lookup"><span data-stu-id="c120b-115">Consequently, `Thread.CurrentPrincipal` is present when it's first checked but is null after the call to `await Task.Yield()`.</span></span>

<span data-ttu-id="c120b-116">Získávání aktuálního uživatele identity z kolekce služby DI aplikace je více možností intenzivního testování, příliš, vzhledem k tomu, že testovací identity můžete snadno vložit.</span><span class="sxs-lookup"><span data-stu-id="c120b-116">Getting the current user's identity from the app's DI service collection is more testable, too, since test identities can be easily injected.</span></span>
