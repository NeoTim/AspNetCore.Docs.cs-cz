## <a name="authentication-library"></a><span data-ttu-id="c7d24-112">Knihovna ověřování</span><span class="sxs-lookup"><span data-stu-id="c7d24-112">Authentication library</span></span>

* <span data-ttu-id="c7d24-121">Použití protokolu založeného na tokenu nabízí menší prostor pro útoky, protože tokeny se neodesílají v rámci všech požadavků.</span><span class="sxs-lookup"><span data-stu-id="c7d24-121">Using a token-based protocol offers a smaller attack surface area, as the tokens aren't sent in all requests.</span></span>

* <span data-ttu-id="c7d24-126">Tokeny mají krátkou životnost, jednu hodinu ve výchozím nastavení, což omezuje okno útoku.</span><span class="sxs-lookup"><span data-stu-id="c7d24-126">Tokens have a short lifetime, one hour by default, which limits the attack window.</span></span> <span data-ttu-id="c7d24-127">Tokeny je také možné kdykoli odvolat.</span><span class="sxs-lookup"><span data-stu-id="c7d24-127">Tokens can also be revoked at any time.</span></span>
* <span data-ttu-id="c7d24-128">Samostatná JWTs nabízí záruky pro klienta a server o procesu ověřování.</span><span class="sxs-lookup"><span data-stu-id="c7d24-128">Self-contained JWTs offer guarantees to the client and server about the authentication process.</span></span> <span data-ttu-id="c7d24-129">Například klient má prostředky ke zjištění a ověření, že tokeny, které obdrží, jsou legitimní a byly vygenerovány jako součást daného ověřovacího procesu.</span><span class="sxs-lookup"><span data-stu-id="c7d24-129">For example, a client has the means to detect and validate that the tokens it receives are legitimate and were emitted as part of a given authentication process.</span></span> <span data-ttu-id="c7d24-130">Pokud se třetí strana pokusí o přepnutí tokenu uprostřed procesu ověřování, může klient detekovat přepínaný token a vyhnout se jeho použití.</span><span class="sxs-lookup"><span data-stu-id="c7d24-130">If a third party attempts to switch a token in the middle of the authentication process, the client can detect the switched token and avoid using it.</span></span>
* <span data-ttu-id="c7d24-131">Tokeny s protokolem OAuth a OIDC nespoléhají na správné chování uživatelského agenta, aby bylo zajištěno, že je aplikace zabezpečená.</span><span class="sxs-lookup"><span data-stu-id="c7d24-131">Tokens with OAuth and OIDC don't rely on the user agent behaving correctly to ensure that the app is secure.</span></span>
* <span data-ttu-id="c7d24-132">Protokoly založené na tokenech, jako je OAuth a OIDC, umožňují ověřování a autorizaci hostovaných a samostatných aplikací se stejnou sadou bezpečnostních charakteristik.</span><span class="sxs-lookup"><span data-stu-id="c7d24-132">Token-based protocols, such as OAuth and OIDC, allow for authenticating and authorizing hosted and standalone apps with the same set of security characteristics.</span></span>
